Audio Format Support

Server Headers & Byte-Range Test

iOS Safari requires servers to support byte-range (partial content) requests to stream audio. Tests the first MP3 for correct headers.

Direct Playback Test

Tap a play button to attempt native browser audio on that file. This is the most direct test possible.

Video Element Audio Test

iOS sometimes allows MP3 playback via a <video> tag when <audio> is blocked. Tests the first track this way.

External Audio Test

Tests a known-working MP3 from a different server. If this plays but your tracks don't, the issue is GitHub Pages serving. If this also fails, the issue is the iPad itself.
